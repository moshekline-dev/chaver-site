Getting Started: Before You Begin a New Book

Pre-Flight Checklist

  • ☐ Review Organizational Plan
  • ☐ Study Genesis as model (Parts A, B, C)
  • ☐ Ensure DWT templates are set up in Expression Web
  • ☐ Verify master CSS is linked and working
  • ☐ Open Excel database and create new sheet/section for book
  • ☐ Create book directory structure:
    • /[BookName]/
    • /[BookName]/Units/
Always work on one phase at a time. Complete Phase 1 entirely before moving to Phase 2. Each phase builds on the previous one.

Phase 1: Unit Identification (Part A)

Goal: Identify and validate all literary units in the book

Deliverable: Part-A-Units.html

Time Estimate: 2-4 weeks of research

Step 1.1: Create the Part A File

  1. Open Expression Web
  2. Create new page from parts-ABC.dwt template
  3. Save as: /[BookName]/Part-A-Units.html
  4. Set page title: "Units of [BookName]"

Step 1.2: Write the Introduction

Follow the Genesis model. Include:

  • Why units matter: Not arbitrary divisions but observable structures
  • Reading implications: How unit boundaries affect interpretation
  • Methodology: Type A (external markers) vs Type B (internal structure)
  • Overview: Total number of units, general characteristics

Genesis Opening as Template

"Genesis has traditionally been read either as a continuous narrative or divided according to modern chapter divisions—a medieval Christian innovation bearing no relationship to the text's original structure. Neither approach does justice to the sophisticated literary architecture the text itself provides..."

Step 1.3: Explain Boundary Markers

Document the TWO validation systems:

Type A: External Markers
  • Toledot formulas ("these are the generations of...")
  • Death formulas with life summaries
  • Transitional phrases ("after these things")
  • Geographic/genealogical envelopes
  • Speech boundaries (for Deuteronomy)
Type B: Internal Structural Perfection
  • Three-part structures
  • Precise numerical patterns
  • Verbal envelopes/inclusios
  • Thematic progression
  • Chiastic arrangements

Step 1.4: Describe Each Unit

For each unit, write a detailed description including:

  • ☐ Opening boundary: What marks the beginning (formula, phrase, shift)
  • ☐ Content summary: What happens in this unit
  • ☐ Closing boundary: What marks the end
  • ☐ Evidence strength: Very strong / Strong / Moderate-to-strong
  • ☐ Type of evidence: Type A, Type B, or both
  • ☐ Key features: Divine names, verbal patterns, structural elements
  • ☐ Relationships: How it connects to surrounding units

Genesis Unit 1 as Template

"The opening unit of Genesis establishes the cosmos in seven days, beginning with 'In the beginning Elohim created the heavens and the earth' and concluding with the seventh day rest. This unit validates through both external and internal markers (Type A + B). The inclusio of 'the heavens and the earth' at beginning (1:1) and end (2:1), combined with the completion formula 'Thus the heavens and the earth were finished' (2:1), creates a perfect envelope structure..."

Step 1.5: Update the Database

For each identified unit, enter into Excel database:

  • Unit ID (e.g., "Exo-I", "Lev-I")
  • Book name
  • Unit number
  • Biblical reference (chapter:verse range)
  • Brief description
  • Format (will determine in Phase 2)
Save your work frequently. Unit identification requires careful reading and re-reading. Don't rush this foundational phase.

Phase 2: Structure Discovery (Part B)

Goal: Discover and document the book's architectural design

Deliverable: Part-B-Map.html

Time Estimate: 3-6 weeks of analysis

Step 2.1: Create the Part B File

  1. Create new page from parts-ABC.dwt
  2. Save as: /[BookName]/Part-B-Map.html
  3. Set title: "The Architecture of [BookName]: From Unit Boundaries to Literary Map"

Step 2.2: Document the Discovery Process

Critical: Write this as a narrative of discovery, not just results. Follow Genesis model:

  1. Initial Grouping: What patterns emerged first?
  2. Testing Hypotheses: What worked? What didn't?
  3. Breakthrough Moments: When did the structure become clear?
  4. Validation: What confirms the pattern is intentional?

Genesis Discovery Narrative Style

"Once the nineteen units were identified and their boundaries established, the first question was whether these units showed any patterns of grouping or relationship. The most obvious division emerged immediately: Units 1-4 deal with universal, primeval history..."

"However, closer examination revealed that Unit 4 (Babel) functions differently from Units 1-3..."

Step 2.3: Identify Major Structural Elements

Look for:
  • Triadic organization: Do units group in threes?
  • Matrix arrangements: Rows and columns creating 2D structure?
  • Alternating patterns: Themes that alternate between units?
  • Concentric rings: Center-focused organization?
  • Chiastic structures: A-B-B'-A' patterns?
  • Independent units: Units that stand outside regular patterns?

Step 2.4: Analyze Divine Name Distribution

Map where YHWH and Elohim appear:

  • Which units use primarily YHWH?
  • Which use primarily Elohim?
  • Which use both (interface positions)?
  • Does distribution follow structural patterns?
Genesis established the pattern: Row 1 = YHWH (above/transcendent), Row 3 = Elohim (below/immanent), Row 2 = both (interface). Check if your book follows similar patterns.

Step 2.5: Create Visual Representations

Develop diagrams showing:

  • Unit groupings (triads, sets)
  • Matrix layout (if applicable)
  • Concentric structure (if applicable)
  • Cross-references between units

Step 2.6: Update Database with Structure

Add to Excel for each unit:

  • Format (e.g., "2×3", "4×2", "13331")
  • Thread (Horizontal or Vertical)
  • Column position (A, B, C, etc.)
  • Row position (1, 2, 3, etc.)
  • Color (assigned by position)
  • Related units
  • Parallel units
Don't force patterns. If the structure isn't clear, keep analyzing. Valid patterns will have multiple converging lines of evidence, not just one or two coincidences.

Phase 3: Book Commentary (Part C)

Goal: Explore theological and interpretive implications of the architecture

Deliverable: Part-C-Commentary.html

Time Estimate: 4-8 weeks of writing

Step 3.1: Create Part C File

  1. Create from parts-ABC.dwt
  2. Save as: /[BookName]/Part-C-Commentary.html
  3. Title: "[BookName]: Architecture and Meaning"

Step 3.2: Write Opening Statement

Begin by:

  • Referencing Part B (the discovery document)
  • Stating your central argument about the architecture
  • Previewing what this structure reveals

Genesis Opening as Model

"This commentary builds on the architectural analysis in 'Discovering the Map of the Book,' which revealed Genesis's sophisticated structure: nineteen units organized into six triads with interwoven covenant and family tracks. That discovery process showed how the structure works. What follows shows what the structure reveals..."

Step 3.3: Develop Major Themes

Organize commentary around 3-5 major themes revealed by structure:

Possible Themes:
  • Concentric organization and meaning
  • Divine name theology
  • Systematic correspondences between units
  • Alternating tracks or threads
  • Progressive development across the book
  • Book's relationship to Torah-wide themes

Step 3.4: Analyze Corresponding Units

For units in matching positions (same row, same column, parallel structure):

  • Identify shared themes or motifs
  • Note verbal parallels
  • Explain theological significance of correspondence
  • Show how patterns validate the structure

Step 3.5: Write Section Conclusions

Each major section should conclude with:

  • Summary of key insights
  • How this theme contributes to book's overall argument
  • Implications for reading and interpretation

Step 3.6: Write Book-Level Synthesis

Conclude with:

  • What the architecture as a whole communicates
  • How structure embodies theology (not just illustrates)
  • The book's contribution to Torah's overall message
  • Reading strategies enabled by understanding structure
Keep the focus on observable patterns. Your claims should be testable against the text. The structure itself is the evidence.

Phase 4: Unit Commentaries (Part D)

Goal: Create comprehensive commentary for each individual unit

Deliverable: All unit HTML files in /Units/ directory

Time Estimate: 3-6 months (depending on number of units)

Step 4.1: Set Up Unit File

  1. Create new page from unit-commentary.dwt
  2. Save as: /[BookName]/Units/[bookname]-unit-01.html
  3. Use two-digit numbering (01, 02, etc.)

Step 4.2: Complete Unit Header

Fill in all metadata fields:

Unit Title: [Book] Unit [#]: [Descriptive Name]
Structure: [Format] (e.g., 2×3 Regular, 4×2 Irregular)
Biblical Reference: [Book Chapter:Verse-Chapter:Verse]
Theme: [Primary thematic focus]
Position: Row [#], Column [Letter] in book matrix

Step 4.3: Add Book Map Visual

  • Create SVG showing all units in book
  • Highlight current unit in contrasting color
  • Position at top of page for immediate context

Step 4.4: Display Full Unit Text

  • Include complete biblical text
  • Color-code according to patterns from database
  • Present in appropriate table structure (regular/irregular)
  • Add color legend explaining pattern identification

Step 4.5: Follow Taxonomy Structure

Use the Unit Commentary Taxonomy as your guide:

Required Sections:
  1. Opening Hook: Compelling entry point
  2. Unit Context: Position in book architecture
  3. Structural Analysis: Dimensional overview, threads, matrix
  4. Detailed Commentary: Cell-by-cell, patterns, themes
  5. Theological Implications: Divine-human dynamics, deeper meanings
  6. Synthesis: Unit's contribution, relevance, summary

Step 4.6: Analyze Warp and Weft Threads

Weft Threads (Columns):
  • Identify and label each column
  • State thematic principle for each
  • Show progression or relationship between columns
Warp Threads (Rows):
  • Identify and label each row
  • State sequential or hierarchical principle
  • Show movement or development across rows

Step 4.7: Document Color-Coded Patterns

For each pattern color used:

Pattern Type Color What It Shows
Horizontal Parallels Blue Connections across rows
Vertical Threads Gold Column-wise progressions
Chiastic Structures Purple A-B-B'-A' patterns
Closure/Frame Pink Inclusios and envelopes
Internal Parallels Lime Within-unit correspondences

Step 4.8: Write Questions for Study

End each unit with 5-7 thought-provoking questions:

  • Questions that emerge from structural analysis
  • Theological implications to explore
  • Contemporary application possibilities
  • Connections to other biblical texts

Step 4.9: Add Navigation

  • Link to previous unit (if not first)
  • Link to next unit (if not last)
  • Link back to book overview/Parts A-B-C
  • Breadcrumb trail functional
Word count targets: Simple units (2,000 words), Standard units (3,000-5,000 words), Complex central units (up to 8,000 words). Let the unit's complexity determine length, not arbitrary targets.

Setting Up DWT Templates in Expression Web

Creating Your First DWT

  1. Open Expression Web
  2. File → New → Page
  3. Select "HTML" as page type
  4. Design your master layout with common elements:
    • Header with site title/logo
    • Navigation menu
    • Breadcrumb area
    • Main content area
    • Footer
  5. Select areas that should be editable
  6. Format → Dynamic Web Template → Manage Editable Regions
  7. Create editable region named "main-content"
  8. File → Save As → Choose "Dynamic Web Template (*.dwt)"
  9. Save to /Assets/Templates/ directory

Attaching Pages to DWT

  1. Open existing page or create new page
  2. Format → Dynamic Web Template → Attach Dynamic Web Template
  3. Browse to your .dwt file
  4. Expression Web will prompt to match content to editable regions
  5. Confirm attachments
  6. Save page
Once a page is attached to a DWT, you can only edit content within the editable regions. To change the template structure, you must edit the .dwt file itself, then Expression Web will update all attached pages.

Template Hierarchy

Create templates in this order:

  1. site-master.dwt - Most general, used as base
  2. book-master.dwt - Inherits from site-master, adds book navigation
  3. parts-ABC.dwt - Inherits from book-master
  4. unit-commentary.dwt - Inherits from book-master

Working with the Database

Daily Database Workflow

  1. Morning: Open Excel database, review current book sheet
  2. During Work: Update fields as you make discoveries
  3. End of Day: Add completion notes, save backup

Key Fields to Maintain

Field When to Update Purpose
ID Phase 1 Unique identifier
Reference Phase 1 Biblical boundaries
Format Phase 2 Structural dimensions
Row/Column Phase 2 Matrix position
Color Phase 2 Visual display
Divine_Name_Pattern Phase 2-3 Theological analysis
Verbal_Patterns_* Phase 4 Color-coded display
URL_Pattern Phase 4 Web publishing
Use Excel's data validation features to create dropdown lists for recurring fields (Book names, Row numbers, Column letters) to ensure consistency.

Troubleshooting Common Issues

Problem: Can't Identify Clear Unit Boundaries

Solutions:

  • Look for multiple converging markers, not just one
  • Check if apparent boundary is actually internal subdivision
  • Consider if unit is larger than expected
  • Review Type B evidence (internal structure) if Type A markers absent
  • Consult Genesis model for similar situations

Problem: Structure Doesn't Seem to Have Clear Pattern

Solutions:

  • Not all books follow identical patterns (Exodus differs from Genesis)
  • Consider whether book has different organizing principle
  • Look for speech boundaries (especially Deuteronomy)
  • Check if independent units are breaking apparent patterns
  • May need to read by columns rather than rows, or vice versa

Problem: DWT Changes Not Appearing on Pages

Solutions:

  • Save the .dwt file (Expression Web should auto-update)
  • If prompted, click "Yes" to update all attached pages
  • Close and reopen Expression Web
  • Check that page is correctly attached to template
  • Verify editable region names match

Problem: Colors Not Displaying Correctly

Solutions:

  • Verify CSS file is linked in page header
  • Check that class names match exactly (case-sensitive)
  • Clear browser cache
  • Validate HTML (tools → Accessibility Reports)
  • Inspect element in browser to see which styles are applying

Problem: Overwhelmed by Scope of Project

Solutions:

  • Focus on one phase at a time
  • Complete one book fully before starting next
  • Set weekly goals (e.g., "Complete 2 unit commentaries this week")
  • Celebrate milestones (each phase completion, each book completion)
  • Remember: Genesis is your complete model to reference
