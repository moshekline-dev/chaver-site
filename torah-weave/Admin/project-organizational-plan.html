1. Project Structure Overview

The Torah Commentary Project consists of three major tiers:

TIER 1: Torah-Wide Overview

  • General introduction to the five books and their relationships
  • The weave/chariot architecture explanation
  • How the books fit together (horizontal and vertical threads)
  • Reading strategies for multiple contexts

TIER 2: Book-Level Analysis (repeated for each of 5 books)

  • Part A: Identifying Literary Units
  • Part B: Discovering the Book's Structure/Map
  • Part C: Book Commentary (based on the map)
  • Part D: Individual Unit Commentaries (19, 19, 22, 13, 13 units)

TIER 3: Administrative/Support Materials

  • Database tracking (Excel files)
  • Unit commentary taxonomy/template
  • Available materials inventory
  • CSS/DWT standards for Expression Web
  • Workflow documentation

4. Expression Web / DWT Strategy

Create 4 Master Dynamic Web Templates to ensure consistency:

1. site-master.dwt

  • Overall site structure
  • Header with site navigation
  • Breadcrumb trail system
  • Footer with copyright/contact
  • Links to CSS

2. book-master.dwt

  • Inherits from site-master.dwt
  • Book-specific navigation (Parts A, B, C, Units)
  • Book map icon display
  • Links to adjacent books
  • Book-specific color scheme variables

3. unit-commentary.dwt

  • Inherits from book-master.dwt
  • Unit header with metadata (title, structure, reference, theme, position)
  • Previous/Next unit navigation
  • Color-coded text display area
  • Standard section structure per taxonomy template
  • Matrix display containers

4. parts-ABC.dwt

  • Inherits from book-master.dwt
  • For Parts A, B, C pages
  • Consistent scholarly format
  • Navigation between parts

6. Workflow for Completing a Book

Phase 1: Unit Identification

  • Create Part-A-Units.html
  • Follow Genesis model: boundary evidence, unit descriptions
  • Document Type A (external markers) and Type B (internal structure) evidence
  • Update database with unit boundaries and references

Phase 2: Structure Discovery

  • Create Part-B-Map.html
  • Document discovery process like Genesis "Discovering the Map"
  • Identify matrix structure, triads, patterns
  • Analyze divine name distribution
  • Map cross-unit relationships
  • Update database with structural relationships

Phase 3: Book Commentary

  • Create Part-C-Commentary.html
  • Flow directly from Part B structure
  • Explore theological implications of architecture
  • Document cross-unit patterns and meanings
  • Keep scholarly, architectural focus
  • Connect to Torah-wide themes

Phase 4: Individual Unit Commentaries

  • Use unit-commentary.dwt template
  • Follow taxonomy template structure
  • Each unit 2,000-8,000 words (depending on complexity)
  • Include book map with unit highlighted
  • Color-coded unit text from database
  • Matrix display with pattern analysis
  • Theological synthesis and contemporary relevance

7. Naming Conventions

File Names

  • All lowercase with hyphens
  • Book names spelled out: genesis-unit-01.html NOT gen-u1.html
  • Two-digit unit numbers: 01 not 1
  • Parts: part-a-units.html, part-b-map.html, part-c-commentary.html

URL Structure

  • Consistent pattern: /torah-commentary-project/Genesis/Units/genesis-unit-01.html
  • Alternative simpler: /Genesis/genesis-unit-01/

CSS Classes and IDs

  • Semantic naming: .unit-header, .matrix-display, .pattern-blue-horizontal
  • Book prefixes where needed: .genesis-row-1, .exodus-column-a
  • Pattern indicators: .blue-horizontal, .gold-vertical, .purple-chiasm

8. Database Integration

Your Excel database is the single source of truth. Use it to:

Primary Functions

  1. Generate HTML unit pages - Export data to populate templates
  2. Track completion status - Add columns for A/B/C/D completion per unit
  3. Maintain consistency - Single source for:
    • Unit boundaries and references
    • Color schemes and patterns
    • Structural positions (row/column)
    • Cross-references between units
    • Key Hebrew terms
    • Divine name patterns

Consider Adding Sheets

  • Book Status: Tracks A/B/C/D completion for each book
  • Cross-References: Maps unit relationships across books
  • Color Patterns: Documents all color-coding systems
  • Progress Log: Date-stamped completion tracking

10. Quality Checklist for Each Book

Before considering a book "complete," verify:

Content Completion

  • ☐ Part A: Units identified with boundary evidence
  • ☐ Part B: Structural map discovered and documented
  • ☐ Part C: Book commentary flowing from structure
  • ☐ Part D: All unit commentaries following taxonomy

Technical Standards

  • ☐ Database updated for all units
  • ☐ All pages use correct DWT
  • ☐ Navigation links functional
  • ☐ Book map graphics created
  • ☐ Color-coding consistent with database

Content Quality

  • ☐ Cross-references to other books added
  • ☐ Hebrew terms properly transliterated
  • ☐ Divine name patterns documented
  • ☐ Theological insights clearly articulated
  • ☐ Reviewed for conformity to style guide

Integration

  • ☐ Links from Torah Overview functioning
  • ☐ Cross-book connections documented
  • ☐ Part of larger Torah weave narrative
  • ☐ Breadcrumb navigation working
