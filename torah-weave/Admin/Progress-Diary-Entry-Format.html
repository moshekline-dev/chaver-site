Progress Diary Entry Format

Purpose

This document provides the exact HTML structure for adding session summaries to the Progress Diary. Claude will generate entries in this format at the end of each session, ready for you to copy/paste into the diary page.

The HTML Template

Standard Entry Structure:

<div class="entry"> <div class="entry-date">[Month Day, Year] - [Brief Title]</div> <h3>[Session Focus/Achievement]</h3> <p>[Brief narrative summary of what was accomplished]</p> <h4>Work Completed:</h4> <ul> <li>[Specific accomplishment 1]</li> <li>[Specific accomplishment 2]</li> <li>[Specific accomplishment 3]</li> </ul> <h4>Files Created/Modified:</h4> <ul> <li><code>[filename.html]</code> - [what was done]</li> <li><code>[filename2.html]</code> - [what was done]</li> </ul> <h4>Database Updates:</h4> <ul> <li>[Book] Unit [#]: [status change or field updates]</li> </ul> <h4>Next Session Priorities:</h4> <ol> <li>[Top priority task]</li> <li>[Second priority task]</li> <li>[Third priority task]</li> </ol> <h4>Chat Reference:</h4> <p><a href="[chat-url]" target="_blank">View this session's chat</a></p> <h4>Technical Notes:</h4> <ul> <li>[Any important observations or decisions made]</li> </ul> </div>

Required Sections

Every Entry Must Include:

  1. Date and Title (in entry-date div)
  2. Session Focus (h3 heading)
  3. Brief Narrative (opening paragraph)
  4. Work Completed (bulleted list)
  5. Next Session Priorities (numbered list)

Optional Sections (Include If Relevant):

  • Files Created/Modified - When files were saved/changed
  • Database Updates - When database was updated
  • Chat Reference - Link to the chat (always good to include)
  • Technical Notes - Important decisions, observations, or insights
  • Outstanding Issues - Problems to resolve in future sessions

How to Add Entry to Progress Diary

Step-by-Step:

  1. Open Progress-Diary.html in Expression Web or text editor
  2. Find the location right after the opening <body> tag and any header content
  3. Paste the new entry HTML at the top (most recent first)
  4. Save the file
  5. Upload to server via FTP

Where Exactly to Paste:

Look for existing entries in the Progress Diary. Paste your new entry above the most recent existing entry, so the diary shows newest-first order.

Best Practices

Writing Good Diary Entries:

  • Be Specific: "Completed Genesis Unit 7" not just "did some work"
  • Include Context: Why this unit matters in the larger structure
  • Note Decisions: Any choices made that affect future work
  • Track Issues: Problems discovered that need resolution
  • Link Everything: Include chat URL so you can review later
  • Plan Ahead: Clear next steps so future sessions start smoothly

Remember:

The Progress Diary is your continuity system. When starting a new chat weeks later, you (and Claude) will read these entries to understand what was done and what's next. Make entries clear, complete, and useful for your future self.
