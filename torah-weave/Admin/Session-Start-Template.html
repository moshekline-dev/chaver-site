Session Start Template

Purpose

This document provides the exact template for starting each new chat session with Claude. Following this template ensures Claude activates the Interactive Workflow Protocol correctly and maintains continuity from previous sessions.

The Basic Template

Copy/Paste This at the Start of Each Chat:

Follow the Interactive Workflow Protocol.

Today we're working on: [SPECIFIC TASK]

Files available:
□ Database: [YES/NO - if yes, note if uploaded to chat]
□ Progress Diary: [LINK or UPLOADED]
□ Other: [Any other relevant files]

What to Fill In

[SPECIFIC TASK] - Choose One:

  • Unit Commentary: "Genesis Unit 5 commentary"
  • Book Part: "Genesis Part C (Commentary) rework"
  • Multiple Units: "Leviticus Units 10-12"
  • Structure Analysis: "Determining Deuteronomy structure"
  • Database Update: "Updating database for completed Genesis units"
  • CSS/Technical: "Fixing styling issues in unit commentaries"
  • Continue Previous: "Continue where we left off last session"

Database Options:

  • YES - Uploaded: "Database: YES (uploaded to this chat)"
  • YES - Will Upload: "Database: YES (will upload when needed for updates)"
  • NO: "Database: NO (not needed today)"

What Happens After You Send This

Claude's Response:

  • Acknowledges the protocol: "Following the Interactive Workflow Protocol..."
  • Reviews context: Fetches relevant documents (index, progress diary, etc.)
  • Confirms understanding: "I can see we're working on [X]. Ready to proceed?"
  • Suggests approach: Outlines the work plan based on the task

Optional Additions

You Can Add These If Relevant:

  • Specific Questions: "Question: Should we use the envelope structure found in Unit 5?"
  • Constraints: "Note: Need to finish in about 1 hour"
  • Previous Issues: "Reminder: We decided to use 'YHWH Elohim' (not 'Yahweh-Elohim')"
  • Priority Items: "Priority: Database must be updated before ending session"

When NOT to Use This Template

Skip the Template If:

  • Quick Question: Just asking a simple question not related to project work
  • Testing Something: Experimenting with code or formatting
  • General Discussion: Discussing methodology without doing actual work

For these cases, just chat normally. Use the template when you're ready to do structured project work that should be tracked and documented.

Quick Reference Card

Minimal Version (When Claude Already Has Context):

Follow protocol. Working on: [TASK]

Use this shorter version if:

  • Continuing work from earlier today
  • Claude already knows the project status
  • You're in the middle of a multi-session task

Troubleshooting

If Claude Doesn't Follow the Protocol:

  • Remind: "Please follow the Interactive Workflow Protocol as documented"
  • Link: Provide the protocol link:
    https://chaver.com/torah-weave/Admin/Interactive-Workflow-Protocol.html
  • Restart: Begin a new message with the full template

If You Forget What You Were Working On:

  • Check the Progress Diary for last session summary
  • Tell Claude: "Check Progress Diary and tell me what we should work on next"
