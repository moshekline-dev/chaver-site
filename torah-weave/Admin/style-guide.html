Color System

Unit Background Colors

Each unit is assigned a background color based on its position in the book's structure:

Color Hex Visual Used In Meaning/Position
#d4f2ff Genesis, Leviticus, Deuteronomy Light Blue - Column A / Row 1 positions
#e6ffee Genesis, Leviticus, Deuteronomy Light Green - Column B / Covenant track
#d4d4ff Genesis, Leviticus Light Purple - Column C / Row 2 positions
#ffe6e6 All books Light Pink - Special/Independent units
#ffccaa Exodus, Numbers Light Orange - Vertical thread units
#c8e6d2 Exodus, Numbers Sage Green - Tabernacle-related units
#f0f0f0 Leviticus Light Gray - Center/neutral positions

Book Identification Colors

Book Primary Color Hex Code Visual
Genesis Blue #5090f0
Exodus Orange #e89040
Leviticus Green #50c080
Numbers Orange #e89040
Deuteronomy Blue #5090f0
Genesis and Deuteronomy share blue (horizontal thread), while Exodus and Numbers share orange (vertical thread). Leviticus uses green as the intersection point.

Pattern Highlighting System

Critical: These colors highlight verbal patterns, structural relationships, and literary devices WITHIN the unit text. They are applied to spans of Hebrew/English text, not backgrounds.

Established Pattern Colors

Blue - Horizontal Parallels

CSS Class: .horizontal1

Hex: #3399FF

What it shows: Connections across rows in the same column. Parallel structures, repeated phrases, or thematic echoes that run horizontally.

Example: "vayomer Elohim" ... "vayomer Elohim" (repeated formula across creation days)

Teal - Secondary Horizontal

CSS Class: .horizontal2

Hex: teal

What it shows: A second layer of horizontal parallels, when multiple distinct patterns exist in the same unit.

Maroon - Tertiary Horizontal

CSS Class: .horizontal3

Hex: maroon

What it shows: A third layer of horizontal parallels for complex units.

Gold - Vertical Threads

CSS Class: .vertical1

Hex: #C09200

What it shows: Column-wise progressions. Themes or vocabulary that develop vertically down a column (warp thread).

Example: Divine name patterns down Column A, blessing formulas down Column B

Pink - Closure/Frame Patterns

CSS Class: .closure

Hex: #CC0066

What it shows: Inclusios, envelope structures, bookends. Opening and closing phrases that frame a unit or section.

Example: "the heavens and the earth" (1:1) ... "the heavens and the earth" (2:1)

Purple - Primary Chiasm

CSS Class: .ciasm1

Hex: #7030A0

What it shows: Chiastic structures (A-B-B'-A' patterns). Elements that reverse or mirror across the unit center.

Example: Abraham's transformation from accuser to God-fearer

Light Purple - Secondary Chiasm

CSS Class: .ciasm2

Hex: #FB81EC

What it shows: A second chiastic pattern within the same unit, or sub-chiasms within larger structures.

Lime - Internal Parallels

CSS Class: .internalparallel

Hex: lime

What it shows: Within-unit correspondences that don't fit the other categories. Verbal echoes, wordplay, or thematic links internal to the narrative.

Example: "knowing" theme connecting tree of knowledge, sexual knowledge, and murder

Typography Standards

Font Families

body {
    font-family: Georgia, serif;
    /* For body text, commentary */
}

.hebrew-text {
    font-family: 'David Libre', 'Frank Ruehl CLM', 'SBL Hebrew', serif;
    /* For Hebrew biblical text */
}

h1, h2, h3 {
    font-family: Georgia, serif;
    /* Headers match body */
}

code, .verse-number {
    font-family: 'Courier New', monospace;
    /* For code and verse references */
}

Font Sizes

Element Size Line Height
Body text 16px 1.6
H1 (Page title) 32px 1.2
H2 (Major sections) 24px 1.3
H3 (Subsections) 20px 1.4
H4 (Minor headings) 18px 1.4
Biblical text (in tables) 16px 1.5
Highlighted patterns 16px (bold) 1.5

Text Styling Rules

  • Bold: Use for emphasis, pattern highlights, verse numbers, cell subdivisions
  • Italic: Use for Hebrew transliteration, book/unit titles when referenced inline, foreign terms
  • Underlining: Avoid (reserved for hyperlinks)
  • ALL CAPS: Use sparingly, only for acronyms or special emphasis

Table Structures for Units

Standard 2-Column Table

<table class="torah-table two-col">
    <tr>
        <td class="torah-column">
            <div class="torah-text">
                <span class="verse-number">1:1</span>
                In the beginning God created the heavens and the earth.
            </div>
        </td>
        <td class="torah-column">
            <div class="torah-text">
                <span class="verse-number">1:3</span>
                And God said, "Let there be light," and there was light.
            </div>
        </td>
    </tr>
</table>

3-Column Table

<table class="torah-table three-col">
    <tr>
        <td class="torah-column">Column A content</td>
        <td class="torah-column">Column B content</td>
        <td class="torah-column">Column C content</td>
    </tr>
</table>

Irregular Structure (e.g., 13331)

<!-- Row 1: Single column -->
<table class="torah-table single-col">
    <tr>
        <td class="torah-column">Single column content</td>
    </tr>
</table>

<!-- Rows 2-4: Three columns -->
<table class="torah-table three-col">
    <tr>
        <td class="torah-column">Column A</td>
        <td class="torah-column">Column B</td>
        <td class="torah-column">Column C</td>
    </tr>
</table>

<!-- Row 5: Single column -->
<table class="torah-table single-col">
    <tr>
        <td class="torah-column">Single column content</td>
    </tr>
</table>

Table Styling Rules

  • Always wrap text content in <div class="torah-text">
  • Use torah-column class for all table cells
  • Verse numbers in <span class="verse-number">
  • Cell subdivisions (A, B, C) in <span class="section-label">
  • No inline styles except those specified in cleanup guide

Hebrew Text Standards

Hebrew Display

<p class="hebrew-text" dir="rtl">
    בְּרֵאשִׁית בָּרָא אֱלֹהִים אֵת הַשָּׁמַיִם וְאֵת הָאָרֶץ
</p>

Transliteration Standards

  • Format: Italic, lowercase except proper names
  • Example: bereshit, toledot, YHWH
  • Divine names: YHWH (all caps), Elohim (capital E)
  • In tables: Hebrew in <span class="hebrew">

Divine Name Usage

Hebrew Transliteration English When to Use
יהוה YHWH YHWH or "the LORD" Technical analysis, structural discussion
אלהים Elohim Elohim or "God" Technical analysis, structural discussion
יהוה אלהים YHWH Elohim YHWH Elohim Eden narrative, compound name discussion

Complete Code Examples

Example 1: Unit with Multiple Patterns

<table class="torah-table two-col">
    <tr>
        <td class="torah-column">
            <div class="torah-text">
                <span class="section-label">A.</span>
                <span class="verse-number">1:1</span>
                In the beginning 
                <span class="horizontal1">God created</span> 
                <span class="closure">the heavens and the earth</span>.
            </div>
        </td>
        <td class="torah-column">
            <div class="torah-text">
                <span class="section-label">B.</span>
                <span class="verse-number">1:3</span>
                And <span class="horizontal1">God said</span>, 
                "<span class="vertical1">Let there be light</span>," 
                and there was light.
            </div>
        </td>
    </tr>
</table>

Example 2: Color Legend

<div class="color-legend">
    <h3>Pattern Color Guide</h3>
    <ul>
        <li><span class="horizontal1">Blue</span> - Horizontal parallels</li>
        <li><span class="vertical1">Gold</span> - Vertical threads</li>
        <li><span class="closure">Pink</span> - Closure patterns</li>
        <li><span class="ciasm1">Purple</span> - Chiastic structures</li>
        <li><span class="internalparallel">Lime</span> - Internal parallels</li>
    </ul>
</div>

Example 3: Unit Header

<div class="unit-header">
    <h1>Genesis Unit 1: Creation in Seven Days</h1>
    <div class="unit-metadata">
        <p><strong>Structure:</strong> 2×3 Regular</p>
        <p><strong>Biblical Reference:</strong> Genesis 1:1-2:3</p>
        <p><strong>Theme:</strong> Cosmic order through divine speech</p>
        <p><strong>Position:</strong> Row 1, Column A</p>
    </div>
</div>
Always validate your HTML after adding patterns. Nested spans and unclosed tags are the most common errors when adding color-coded patterns.
