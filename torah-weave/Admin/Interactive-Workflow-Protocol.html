2. Token Tracking Protocol (CORRECTED v2.4)

🔴 THE PROBLEM WITH v2.3 AND EARLIER:

Claude sees token counts AFTER each tool call, showing cumulative usage up to that point. However, when Claude generates large content (like a 300-line HTML document), those tokens aren't counted until AFTER the response completes.

Result: A response showing "115,000 tokens (61%)" might actually push the total to 165,000 tokens (87%) after generation completes, causing unexpected session termination.

How Token Counting Actually Works:

What Claude Sees What It Means The Problem
Token usage: 31,620/190,000 Cumulative tokens used SO FAR in conversation Does NOT include tokens about to be generated
Reported at END of response Shows usage BEFORE current response Large outputs aren't counted until too late
Large document = 20-30K tokens Can jump from 60% to 85% in one response No warning before hitting danger zone

The Solution: Conservative Pre-Generation Estimation

New Rule: Claude must estimate token cost BEFORE generating large content and be conservative with thresholds.

Mandatory Token Status Format

At the end of EVERY response, Claude must include:

───────────────────────────────────────
📊 Token Status: [CURRENT] / 190,000 ([%]% used) [STATUS]
[ESTIMATE]: Next response may use ~[X]K tokens
───────────────────────────────────────

Status Thresholds (CONSERVATIVE)

Token Range Percentage Status Action Required
0 - 95,000 0 - 50% ✅ SAFE Work normally
95,001 - 114,000 51 - 60% ⚠️ CAUTION Limit large outputs. Warn before generating 300+ line documents.
114,001 - 133,000 61 - 70% 🟡 WARNING No large document generation. Break work into smaller pieces. Suggest wrapping up soon.
133,001 - 152,000 71 - 80% 🟠 DANGER Stop all content creation. Complete current task only. Create diary entry. Prepare to end session.
152,001+ 80%+ 🚨 CRITICAL IMMEDIATE: Stop work. Create diary entry NOW. End session.

Special Rules for Large Document Generation

Before generating any document over 200 lines:
  1. Check current token count
  2. Estimate document will use 15-30K tokens
  3. Calculate projected total: current + 30K
  4. If projected total exceeds 114K (60%), STOP and warn user first

Example Token Warning (Before Large Output):

⚠️ TOKEN WARNING

Current usage: 98,000 / 190,000 (51.6%)
Estimated next response: ~25,000 tokens (full document rewrite)
Projected total: ~123,000 (65%)

This will move us into WARNING zone.

Options:
A. Proceed with full document (risk session ending)
B. Generate in sections (safer, takes longer)
C. Defer to next session

Which would you prefer?
